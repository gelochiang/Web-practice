初始設定 ：
設定自己的名稱、電子郵件
git config --global user.name "<使用者名字>"
git config --global user.email "<電子信箱>"
設定Git輸出(output)結果的顏色：
git config --global color.ui auto
alias為Git的命令設定別名。ex:「checkout」可以省略為「co」來執行 :
git config --global alias.co checkout
新建repository :
建立一個work的目錄，並將目錄create for local repository
mkdir work #建立一個work的目錄
cd work #取得work的URI
git init
#移動到這目錄並create for local respository
提交檔案 :
首先在work裡加入king.txt（下方是裡面的內容）
你好我是在童話故事裡出現的脫衣國王
運用status來確認工作目錄與索引的狀態 :
git status
因為king.txt還不是歷史追蹤的對象，必需加入到索引來成功變成指定追蹤的對象（下方有兩種不一樣的處理方式）
git add ... # add為指定
至於要追蹤全部的檔案：
git add *
git add . #和上面不同的是把file變成「.」表示目錄下的所有檔案
可與上面的status結合，檢查是否已經加入索引 :
git add sample.txt
git status
> #會出現先前更改了什麼，可以準備commit，假如沒改變會顯示沒有東西能上傳
確認後用commit把檔案傳送出去 :
git commit -m "提交訊息"
#提交訊息就像是註解你裡頭改了什麼，讓其他人理解
接著能用log來看歷史紀錄 :
git log
小秘密：
在安裝的同時也會安裝gitk的工具，可用圖形化得知歷史紀錄（美侖美奐）
gitk
Push到遠端數據庫（上傳king.txt) : 首先要先指定增加這個遠端數據庫，和前面的很類似，會用到name/url
git remote add (name) (url)
為了讓後續更方便找到這個repository可以把它命名為origin
git remote add origin (這裡輸入 remote 的 URL)
當要push變更到remote的時候，會用到push的命令，一樣會用到repository的地址，然後選擇要push的分支（這方面之後再更細研究）
git push (repository) (refspec)...
在提交的時候加入-u可以之後省略指定分支的名稱，但push到空白的remote時就一定要加入分支名稱，不能省略
git push -u origin master
Clone遠端數據庫（複製）: 要先得知repository的name和directory才能確切知道要從哪裡clone
git clone (url)
可以試著複製到另一個目錄裡，看有沒有包含剛剛push上去的king.txt(當然裡面也會出現剛剛的內容）
你好我是在童話故事裡出現的脫衣國王
在clone的repository執行push
在剛剛clone下來的king.txt檔上加入下方文字
因為真的天氣熱所以才脫的
git add king.txt
git commit -m "脱衣的原因" # “脱衣的原因”是註解
git push
從remote執行pull(同步remote的修改內容）
git pull
合併修改記錄 在執行push的時候，如果有人已經先更新了remote，而自己沒有更新自己本地的repository，自己的push就會遭到拒絕。這時候就要合併操作導入其他人的修改歷史，才可以成功的push(所謂先下手為強XD)
通常git會自動的合併，但自己修改的地方剛好和上次更新的地方一樣就要改成手動合併了，修改完成用commit提交。
最後可以用log來確認自己的修改紀錄（加入－－graph可以圖表方式顯示分支歷史，加入－－oneline簡化輸出訊息
git log --graph --oneline
